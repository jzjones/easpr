easpr-elements

This repository contains the custom WebRTC elements created by the Easpr team. To see a working demo of these elements in action, please visit here. To see the demo's source, go here.

Installing the elements

To use these elements, install them with Bower using the following command: 
bower install easpr-elements
This will provide each of the elements as well as their dependencies in a folder called bower_components that is placed in the current directory.

Using the elements

To use an element in your HTML page, first import Polymer's platform.js: 
<script src='bower_components/platform/platform.js'></script>
Then, you are free to import the element with the following line: 
<link rel='import' href='bower_components/easpr-elements/element-name/element-name.html'>
This allows you to place the element in your document like any other tag, like so: 
<element-name></element-name>

Attribution

These elements were created using Polymer, PureCSS, LeafletJS and RTCMultiConnection, a WebRTC library. In addition, packaging was done by using Bower.

Element documentation

webrtc-element

This is the 'base' WebRTC custom element that is extended by each of the remaining elements. As such, it is not intended to be used by itself, though it does need to be installed for the other elements to work.

Attributes:

firebaseURL - The URL for the Firebase used as a signalling server.
Does not need to be imported or instantiated.


webrtc-container

This element extends webrtc-element and functions as a container for the videochat, textchat and map elements. This element can be used to add the functionality of the other elements with an attrative layout easily.

Attributes:

sessionName - Takes a string to specify the name of the WebRTC session.
firebaseURL - The URL for the Firebase used as a signalling server.

Import code:

<link rel='import' href='bower_components/easpr-elements/webrtc-container/webrtc-container.html'>

Placement code:

<webrtc-container></webrtc-container<


videochat-element

This element extends webrtc-element and provides conference call functionality.

Attributes:

sessionName - Takes a string to specify the name of the WebRTC session.
firebaseURL - The URL for the Firebase used as a signalling server.

Import code:

textchat-element

This element extends webrtc-element and provides peer-to-peer text communication to all within the same session.

Attributes:

sessionName - Takes a string to specify the name of the WebRTC session.
firebaseURL - The URL for the Firebase used as a signalling server.

Import code:

<link rel='import' href='bower_components/easpr-elements/textchat-element/textchat-element.html'>

Placement code:

<textchat-element></textchat-element>


map-element

This element extends webrtc-element and provides collaborative map functionality. Clicking on the map will place a marker that will then be seen by all in the current session.

Attributes:

sessionName - Takes a string to specify the name of the WebRTC session.
firebaseURL - The URL for the Firebase used as a signalling server.

Import code:

<link rel='import' href='bower_components/easpr-elements/map-element/map-element.html'>

Placement code:

<map-element></map-element>
